# Feature Specs

Novas features planejadas com `tlc-spec-driven` devem nascer aqui.

Estrutura esperada por feature:

- `spec.md`
- `context.md` quando houver decisoes abertas capturadas com o usuario
- `design.md` para mudancas grandes/complexas
- `tasks.md` quando a execucao exigir decomposicao formal

Nao use este diretorio para copiar secrets, logs sensiveis ou dumps de `runtime/config`.
